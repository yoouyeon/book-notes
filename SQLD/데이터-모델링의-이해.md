## 데이터모델의 이해

### 데이터 모델링의 세 가지 관점

1. 데이터 관점 : 업무가 어떤 데이터와 관련이 있는지, 또는 데이터 간의 관계는 무엇인지에 대해서 모델링 하는 방법 (What, Data)
2. 프로세스 관점 : 업무가 실제로 하고 있는 일은 무엇인지, 또는 무엇을 해야 하는지를 모델링 하는 방법 (How, Process)
3. 데이터와 프로세스의 상관 관점 : 업무가 처리하는 일의 방법에 따라 데이터는 어떻게 영향을 받고 있는지 모델링하는 방법 (Interaction)

### 데이터 모델링의 세 가지 중요 개념

- 업무가 관여하는 어떤 것 (Things)
- 업무가 관겨하는 어떤 것의 성격 (Attributes)
- 업무가 관여하는 어떤 것의 관계 (Relationships)

### 논리적 데이터 모델링

논리적 데이터 모델링의 결과로 얻어지는 논리 데이터 모델은 데이터 모델링이 최종적으로 완료된 상태이다. 물리적인 스키마를 설계하기 전 단계이다.

## 엔터티

현실 세계의 개체를 데이터베이스에서 표현하는 논리 단위
반드시 하나 이상의 속성을 가져야 한다.
최소 2개 이상의 인스턴스를 보유해야 한다.
최소 1개 이상의 관계를 가져야 한다. (통계성, 코드성 엔터티는 제외)
엔터티 이름은 전체 도메인 모델에서 유일해야 한다.

### 엔터티의 명명 기준

- 업무 용어 기반의 단수형 명사를 사용한다.

## 속성

### 엔터티와 속성

- 한 개의 엔터티는 두 개 이상의 인스턴스 집합이어야 한다.
- 하나의 속성은 하나의 속성값을 가져야 한다. (하나 이상의 속성값을 갖게 되는 경우 정규화가 필요하다)
- 한 개의 엔터티는 두 개 이상의 속성을 갖는다
- 엔터티 하나의 인스턴스는 다른 엔터티의 인스턴스 간 관계인  Pairing을 갖는다.

### Domain(도메인)

Domain(도메인) : 속성에 대한 값의 범위 같은 제약사항을 ㅣ기술한다.

## 관계

### 관계 정의 시 체크 사항

관계를 정의할 때 주요하게 체크해야 하는 사항

- 두 개의 엔터티 사이에 관심 있는 연관규칙이 존재하는가
- 업무기술서, 장표에 관계연결을 가능하게 하는 동사(Verb)가 있는가?
- 업무기술서, 장표에 관계연결 규칙이 서술되어 있는가?
- 두 개의 엔터티 사이에 정보의 조합이 발생되는가?

### ERD 관계

### ERD 작성 순서

관계의 참여도 다음에 필수 여부를 기술한다는 것을!!! 그만 좀 헷갈리자. (참필…챔필…)

```sql
1. 엔터티를 그린다
2. 엔터티를 적절하게 배치한다
3. 엔터티 간의 관계를 설정한다
4. 관계명을 기술한다.
5. 관계의 참여도를 기술한다.
6. 관계의 필수 여부를 기술한다.
```

### Relationship Degree/Cardinality (관계 차수)

엔터티 간 1:1, 1:M 과 같이 관계참여 인스턴스의 수를 지칭하는 것

### 무결성 제약조건

- 도메인 무결성 제약조건
- 참조 무결성 제약조건

## 식별자

### 키의 종류

슈퍼키 : 유일성만 만족하면 된다. 따라서 유일한 인스턴스를 식별할 수 있는 속성 조합이라면 모두 슈퍼키가 될 수 있다.

후보키 : 유일성과 최소성을 모두 만족하는 속성

주키 : 후보키 중 선택된 하나의 키

대체키 : 후보키 중 주키가 아닌 나머지

### 식별 관계

부모 테이블의 식별자(Primary Key)가 자식 테이블의 식별자의 일부로 사용되는 관계를 의미한다.

자식 테이블의 식별자는 NOT NULL이며 UNIQUE한 특성을 갖기 때문에 해당 값은 반드시 입력되어야 한다.

- 자식 주식별자 구성에 부모 주식별자를 포함해야 한다.
- 강한 연결 관계를 의미한다.
- 부모 엔터티의 주식별자 속성을 자식 엔터티의 식별자에 포함한다.

IE 표기법에서는 두 엔터티 사이의 관계선을 실선으로 표현
Barker 표기법에서는 자식 엔터티 쪽에 bar(|) 기호로 표현

IE 표기법에서 비식별관계는 점선으로 표현한다.

### 비식별자 관계 (Non-Identifying Relationship)

부모 엔터티로부터 속성을 받았지만 자식 엔터티의 주식별자로 사용하지 않고 일반적인 속성으로만 사용하는 것

비식별관계

- 부모 쪽의 관계 참여가 선택 관계이다.
