## 정규화

**정규화** : 데이터의 중복을 제거하고, 삽입·삭제·갱신 시 발생할 수 있는 이상 현상(Anomaly)을 방지해서 데이터의 무결성을 확보하기 위해 수행한다. 테이블을 속성 기준으로 분해하는 과정에서 조인 수가 증가해 조회 성능이 저하될 수 있다.

**3차 정규화** : 식별자를 제외한 일반 속성 간의 종속을 제거한다.

**반정규화**

- 테이블 반정규화, 칼럼 반정규화는 데이터 무결성에 영향을 미친다.
- 관계의 반정규화 기법 중 중복 관계 추가는 데이터 무결성을 깨뜨릴 위험 없이 데이터 처리 성능을 향상시킬 수 있다.
- 관계의 반정규화가 유용한 경우 : 데이터 모델 전체가 관계로 연결되어 있고, 서로 먼 관계 간의 조인이 빈번하게 발생해 성능 저하가 예상되는 경우

## 모델이 표현하는 트랜잭션의 이해

**트랜잭션의 특성 (ACID)**

- **원자성(Atomicity)** : 트랜잭션 내의 모든 작업은 모두 수행되거나 모두 수행되지 않아야 한다.
- **일관성(Consistency)** : 트랜잭션 수행 전과 후에 데이터는 항상 일관된 상태를 유지해야 한다.
- **고립성(Isolation)** : 동시에 실행되는 트랜잭션은 서로 영향을 주지 않아야 한다.
- **지속성(Durability)** : 트랜잭션이 커밋된 후에는 그 결과가 영구적으로 반영되어야 한다.

## NULL 속성의 이해

**`NULL` 비교** : `NULL`과 `NULL`은 비교할 수 없다. `NULL = NULL`은 `TRUE`가 아닌 `UNKNOWN`을 반환한다. `WHERE` 절은 `TRUE`인 행만 반환하므로 `WHERE NULL = NULL`은 아무 행도 반환하지 않는다.

**`IS NULL` / `IS NOT NULL`** : `NULL` 여부를 확인할 때는 `=` 연산자 대신 `IS NULL`, `IS NOT NULL`을 사용해야 한다.

**`NULL` 연산** : `NULL`에 어떤 값을 연산해도 결과는 항상 `NULL`이다.

```sql
-- 아래는 모두 NULL을 반환한다
SELECT NULL * 2
SELECT NULL * 3
SELECT NULL * NULL
SELECT NULL + 100
```

**SQL Server와 Oracle의 `NULL` 인덱스 저장 방식 차이**

- SQL Server는 `NULL` 값을 인덱스 맨 앞에 저장한다. Oracle은 맨 뒤에 저장한다.
- SQL Server는 인덱스 구성 칼럼이 모두 `NULL`인 레코드도 인덱스에 저장한다.
- Oracle은 인덱스 구성 칼럼 중 하나라도 `NULL`이 아닌 레코드만 인덱스에 저장한다.
