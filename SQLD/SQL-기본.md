## SQL 문법 구조

```sql
SELECT <어쩌고>
FROM <어쩌고>
WHERE <어쩌고>
```

## `NVL`

`NVL (대상 컬럼, 대상 컬럼의 값이 NULL이면 반환할 값)`

## 집계 함수와 `WHERE` 절

`AVG`와 같은 집계 함수는 `WHERE` 절에서 쓸 수 없다.

이유 : 집계함수는 `GROUP BY`가 선행되어야 쓸 수 있다. 그런데 `WHERE` 절은 `GROUP BY`보다 먼저 실행됨.

집계함수를 이용해서 조건을 주기 위해서는 `GROUP BY` 이후에 실행되는 `HAVING` 절을 이용하면 된다.

## FULL OUTER JOIN

양쪽으로 OUTER JOIN을 하는 것이다.
내부적으로 LEFT JOIN과 RIGHT JOIN을 하는 것.
조인 조건이 있어야 한다.
오라클에서는 (+) 기호를 양쪽에서 사용할 수 없다.

## Sort Merge 방식의 조인

두개의 테이블을 정렬한 뒤에 Merge하는 방식이다. 완료되면 한번의 Full Scan으로 데이터를 검색

기본키와 외래키 관계에서 외래키에 인덱스가 없을 때 효율적인 방식이라서 이런 상황에서 옵티마이저가 Sort Merge 방식으로 동작하게 한다.

## 분산 데이터베이스 특징

- 지역 자치성 (P)
- 점증석 시스템 용량 확장이 가능함 (P)
- 빠른 응답 속도 (P)
- 오류의 잠재성이 높음 (N)
- 처리 비용이 증가함 (네트워크 구축...) (N)

## SQL의 실행 계획

SQL문의 처리를 위한 절차와 방법을 표현하는 것이다.
액세스 기법, 조인 순서, 방법 같은 것들이 포함됨.
실행 계획이 달라도 결과가 달라지지는 않지만 성능의 차이는 있다.
최적화 정보란 실행 계획의 단계 별 예상 비용을 의미한다.

## 데이터베이스와 테이블

데이터베이스에는 N개의 테이블을 만들 수 있다. 데이터의 특성에 따라서 하나가 좋을 수도, 여러 개가 좋을 수도 있다.
모든 자료들은 테이블에 입력되고, 테이블에서 조회하고, 수정하고, 삭제할 수 있다.

## `WHERE` 절의 `AND`, `OR`

`WHERE` 절에서 `AND`와 `OR` 가 함께 사용되면 **`AND`가 먼저 평가된다.**

```sql
WHERE GRADE='A' OR GRADE='B' AND STATUS='ACTIVE'
```

GRADE가 'A' 인 데이터
또는
GRADE가 'B'이고 STATUS가 'ACTIVE'인 데이터

## SQL 처리흐름도

실행 계획을 시각화한 것 (SQL의 처리 절차를 시각적으로 표현한 것)
인덱스 스캔, 전체 테이블 스캔 같은 액세스 기법을 표현할 수 있다.
성능적인 측면도 함께 표현할 수 있음
