## 관계형 데이터베이스 개요

**데이터베이스와 테이블** : 데이터베이스에는 N개의 테이블을 만들 수 있다. 데이터의 특성에 따라서 하나가 좋을 수도, 여러 개가 좋을 수도 있다. 모든 자료들은 테이블에 입력되고, 테이블에서 조회하고, 수정하고, 삭제할 수 있다.

**분산 데이터베이스의 특징**

**장점**
- 지역 자치성
- 점증적 시스템 용량 확장이 가능
- 빠른 응답 속도

**단점**
- 오류의 잠재성이 높음
- 처리 비용이 증가함 (네트워크 구축 등)

## SELECT 문

**SQL 기본 구조**

```sql
SELECT <컬럼>
FROM <테이블>
WHERE <조건>
```

**`DISTINCT`** : 중복된 행을 제거하고 한 번만 출력한다.

## 함수

**`NVL(대상 컬럼, NULL일 때 반환할 값)`** : 대상 컬럼의 값이 NULL이면 두 번째 인자를 반환한다.

**`NEXT_DAY(날짜, 요일)`** : 해당 날짜 이후 첫 번째로 오는 지정 요일의 날짜를 반환한다.

**`ADD_MONTHS(날짜, 개월수)`** : 해당 날짜에 N개월을 더한 날짜를 반환한다.

**날짜 연산** : Oracle에서 날짜형에 숫자를 더하면 일(day) 단위로 인식한다.

> SYSDATE가 2025-06-09 12:12:12라면 SYSDATE + 1은 2025-06-10 12:12:12

- `1/24` : 1시간
- `1/24/60` : 1분
- `1/24/60/60` : 1초

**`SUBSTR(문자열, 시작위치, 추출할 글자 수)`** : 문자열에서 시작위치부터 지정한 길이만큼 잘라서 반환한다. 시작위치는 1부터 시작하고, 길이를 생략하면 시작위치부터 끝까지 반환한다.

> `SELECT SUBSTR('DATABASE', 1, 4)` → DATA

**`ROUND(값, 자릿수)`** : 값을 지정한 자릿수로 반올림해서 반환한다.

- 자릿수가 양수 : 소수점 N번째 자리까지 반올림 → `ROUND(456.789, 1)` = 456.8
- 자릿수가 0 : 정수로 반올림 → `ROUND(456.789, 0)` = 457
- 자릿수가 음수 : 정수부 N번째 자리에서 반올림 → `ROUND(456.789, -2)` = 500

## WHERE 절

**`AND`, `OR` 우선순위** : `WHERE` 절에서 `AND`와 `OR`가 함께 사용되면 `AND`가 먼저 평가된다.

```sql
WHERE GRADE = 'A' OR GRADE = 'B' AND STATUS = 'ACTIVE'
-- GRADE가 'A'인 데이터
-- 또는 GRADE가 'B'이고 STATUS가 'ACTIVE'인 데이터
```

**`<>`** : 같지 않다는 의미의 비교 연산자

## GROUP BY, HAVING 절

**집계 함수** : `COUNT`, `SUM`, `AVG`. `NULL`을 제외하고 집계한다.

**`HAVING` 절** : 집계 함수는 `WHERE` 절에서 사용할 수 없다. `WHERE` 절은 `GROUP BY`보다 먼저 실행되기 때문이다. 집계 함수를 조건으로 사용하려면 `GROUP BY` 이후에 실행되는 `HAVING` 절을 사용해야 한다.

## ORDER BY 절

**기본 정렬 순서** : 정렬 기준을 생략하면 기본값으로 ASC(오름차순) 정렬된다.

**정렬 기준 표기** : 컬럼명 대신 `SELECT` 절에 기술한 컬럼의 순서 번호나 ALIAS명을 사용할 수 있다.

## 조인

**조인 조건** : 여러 테이블로부터 원하는 데이터를 조회할 때, 테이블이 N개이면 최소 N-1개의 조인 조건이 필요하다.

**조인 유형**

- **EQUI JOIN** : 조인 컬럼이 1:1로 매핑 가능한 경우 사용한다.
- **NON-EQUI JOIN** : 등가 조건이 성립하지 않는 테이블 간에 조인을 거는 방법이다.
- **OUTER JOIN** : 조인 조건을 만족하지 않는 데이터도 조회할 수 있다.
- **SELF JOIN** : 하나의 테이블을 논리적으로 분리시켜 EQUI JOIN을 수행한다.
- **Natural 조인** : 두 테이블에서 이름이 같은 컬럼을 기준으로 자동으로 조인한다.

**Cartesian Product / CROSS JOIN** : 조인 조건이 없으면 두 테이블의 모든 행을 조합한 카테시안 곱이 발생한다. 결과 행 수는 두 테이블 행 개수의 곱이다.

```sql
-- Oracle 방식
SELECT * FROM TAB1, TAB2 ORDER BY 1;

-- ANSI 방식 (CROSS JOIN은 ON 절을 사용하지 않는다)
SELECT * FROM TAB1 CROSS JOIN TAB2 ORDER BY 1;
```

**Oracle-style 조인** : 테이블을 FROM에 콤마로 나열하고 조인 조건을 WHERE에 적는 방식이다.

**ANSI 조인** : `JOIN ... ON ...` 형태로 조인 조건을 명확하게 표현하는 표준 SQL 조인 방식이다.

**테이블 ALIAS** : FROM 절에서 테이블에 ALIAS를 지정한 경우, SELECT 절에서는 반드시 테이블명이 아닌 ALIAS를 사용해야 한다.

## 표준 조인

**LEFT OUTER JOIN** : 왼쪽 테이블의 행은 전부 유지하고, 오른쪽 테이블에서 조건이 맞는 행을 붙인다. 조건에 맞는 행이 없으면 오른쪽 컬럼은 `NULL`로 채운다.

```sql
FROM T1
LEFT OUTER JOIN T2
  ON T1.COL = T2.COL
```

**FULL OUTER JOIN** : LEFT JOIN과 RIGHT JOIN을 합친 것으로, 양쪽 테이블의 행을 모두 유지한다. 조인 조건이 필요하며, Oracle에서는 `(+)` 기호를 양쪽에 동시에 사용할 수 없다.
