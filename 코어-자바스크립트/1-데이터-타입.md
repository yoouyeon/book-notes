# 1 데이터 타입

## 자바스크립트의 데이터 타입

- **원시형 (primitive type)** : 숫자, 문자열, 불리언, null, undefined, Symbol (ES6)
- **참조형 (reference type)** : 객체 (Date, RegExp, Array, …, Map, Set, WeakMap, WeakSet) (ES6)

값을 복제할 때

- 원시형은 **값이 담긴 주솟값**을 복제한다.
- 참조형은 **값이 담긴 묶음의 주솟값**을 복제한다.

## 메모리 배경지식

**비트(bit)**는 0과 1중 하나를 표현할 수 있는 메모리 조각이다. 비트는 너무 작은 단위이기 때문에 메모리 안에서 비트 단위로 관리하면 비효율적이다. 따라서 메모리와 관리비용 모두를 만족시킬 수 있는 적당한 단위를 찾았고, 그 단위가 **바이트(byte)**이다.

**바이트**는 8bit를 한 묶음으로 하는 단위이다. 1bit는 2<sup>1</sup> = 2개의 데이터를 표현할 수 있고, 1byte는 2<sup>8</sup> = 256개의 데이터를 표현할 수 있다.

비트는 메모리 상에서 고유한 식별자를 갖는다. 바이트 역시 시작하는 비트의 식별자를 이용해서 정확한 위치를 파악할 수 있다. 이렇게 메모리 상에서 데이터를 찾을 수 있는 식별자를 **메모리 주솟값**이라고 한다.

## 데이터 타입 배경지식

C/C++, 자바 같은 정적 타입 언어는 메모리를 최대한 효율적으로 사용하기 위해서 타입별로 할당할 메모리를 미리 지정해두었다. 예를 들어서 같은 숫자라도 short 타입은 2byte, int 타입은 4byte를 할당하도록 해서 필요한 수의 범위를 보고 골라서 할당할 수 있다.

하지만 자바스크립트는 메모리 용량이 비교적 넉넉한 시대에 등장했기 때문에 메모리 공간을 상대적으로 널널하게 할당한다. 따라서 예를 들어서 숫자의 경우에 정수형인지, 실수형인지를 구분하지 않고 8바이트를 확보한다.

> 변수와 식별자
> 
> - **변수(variable)** : 변할 수 있는 무언가 (데이터)
> - **식별자(identifier)** : 데이터를 식별하는 데 사용하는 이름. 변수명

## 변수 선언과 값 할당

```javascript
var name; // 1
name = '정대만'; // 2, 3
```

1. 메모리 어딘가에 공간을 할당하고, 그 공간의 식별자를 `name`으로 한다.
2. 비어있는 메모리 공간(@5000)에 `“정대만”`이라는 문자열을 저장한다.
3. `name` 공간에 @5000을 저장한다.

이렇게 변수와 데이터를 별도의 공간에 저장하는 이유는 데이터 변환을 자유롭게 하고 메모리를 효율적으로 관리하기 위함이다. `name`은 문자열을 담는 변수가 아닌 그냥 변수이기 때문에 “정대만”을 담을 수도 있고 숫자 0을 담을 수도 있다. 정적 타입 언어처럼 형변환을 거칠 필요가 없다는 것이다. 또한 같은 데이터를 많은 변수에 저장해야 할 때 변수 하나하나마다 그 데이터를 할당하는 것 보다는 다른 공간에 데이터를 저장해두고 그 주소값만 각 변수에 할당해주는 것이 더 메모리 효율이 높기도 하다.

- 변수와 상수 : 재할당이 가능한지/불가능한지 ⇒ 변수 영역의 문제
- 가변과 불변 : 값을 변경할 수 있는지 ⇒ 데이터 영역의 문제

## 기본형 데이터는 불변하다

```javascript
var name;
name = '정대만';
name = '불꽃남자 ' + name;
```

이 코드를 실행하면 총 메모리 공간에 “정대만” 과 “불꽃남자 정대만” 두가지 문자열이 할당된다. 2번째 라인에서 “정대만”이 담긴 주솟값이 name에 할당되고, 3번째 라인에서 기존 name에 담겨 있던 “정대만” 이 수정되는 게 아니라 새로운 문자열 “불꽃남자 정대만”이 생성되어 메모리에 저장되고, 그 주솟값이 다시 name에 할당된다.

이렇게 한 번 만들어지면 변경이 새로 만들어지는 동작을 통해서만 이루어지는 것이 불변값의 성질이고, 이렇게 만들어진 값은 가비지 컬렉팅을 당하지 않는 한 변하지 않는다.

## 참조형 데이터는 기본적으로 가변하다

```javascript
var player = {
  name: '정대만',
  number: 4,
};
player.number = 14;
```

1. 메모리 어딘가에 공간을 할당하고, 그 공간의 식별자를 player로 한다.
2. 비어있는 메모리 공간에 여러 개의 프로퍼티를 담을 수 있는 공간을 마련한다. (@3000)
3. player 공간에 @3000을 저장한다.
4. @3000 공간 어딘가에 name, number 이름을 저장한다.
5. 비어있는 메모리 공간에 “정대만”과 4를 저장하고 name과 number에 각각 할당한다.

변수 player는 가변하기 때문에 내부 프로퍼티의 값을 변경할 수 있다. `player.number = 14`를 실행하면, player 객체 자체의 프로퍼티 값이 변경된다.

- 얕은 복사 : 바로 아래 단계의 값만 복사하는 것
- 깊은 복사 : 내부의 모든 값들을 하나하나 전부 복사하는 것

참조형 데이터에서 얕은 복사를 수행하면 원본과 사본이 동일한 데이터를 가리키게 된다.

## 깊은 복사 구현하는 법

1. 재귀
2. `JSON.parse(JSON.stringify(target))` ⇒ JSON으로 변경할 수 없는 프로퍼티는 무시함

JS 엔진이 undefined를 반환하는 경우 : 사용자가 값을 지정할 것 같은 상황인데 지정하지 않은 경우

1. 값을 대입하지 않은 변수에 접근할 때
2. 객체 내부에 존재하지 않는 프로퍼티에 접근할 때
3. return문이 없거나 호출되지 않는 함수의 실행 결과

사용자가 직접 값이 없음을 표현하고 싶다면 undefined보다는 null을 사용하는 것이 더 명시적이다.

typeof null이 object라는 것에 주의 (참고: [The history of “typeof null”](https://2ality.com/2013/10/typeof-null.html)) 그래서 어떤 변수의 값이 null인지를 확인하기 위해서는 typeof 대신 동등연산자/일치연산자를 활용해야 한다.
