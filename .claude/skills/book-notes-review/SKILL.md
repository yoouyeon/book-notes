## Derived Values

- 노트 파일 경로 : $ARGUMENTS[0]
- 현재 날짜 : !`date +"%Y-%m-%d"`

## Steps

`반복 횟수`를 1로 초기화한다.
각 반복 시작 시 아래 체크리스트를 복사하여 진행 상황을 추적한다:

```
검토 진행 상황 (n/5회):
- [ ] 파일 읽기
- [ ] 오타 탐지
- [ ] 사실 오류 탐지
- [ ] 결과 제시 및 사용자 응답 대기
```

### Step 1. 파일 읽기

노트 파일을 읽는다. 매 반복마다 새로 읽는다.

### Step 2. 오타 탐지 및 수정

오타를 찾아 모두 수정하고 수정 내역을 알린다.

### Step 3. 사실 오류 탐지

사실과 다른 부분을 찾아 올바른 내용과 함께 목록으로 정리한다.
내용 수정은 사용자가 직접 수행한다. (AI가 절대 직접 수정하지 않는다.)

### Step 4. 결과 제시

사실 오류 목록을 제안한다.

- 오타도 없고 사실 오류도 없는 경우 → Finalize로 이동한다.
- 사실 오류 없이 오타만 수정한 경우 → Finalize로 이동한다.
- 사실 오류가 있는 경우 → 직접 수정해달라고 안내하고 Step 5로 이동한다.

### Step 5. 사용자 응답 처리

사용자가 수정 완료를 알리면 다음과 같은 판단 조건으로 진행 방향을 결정한다.

- 사용자가 중단을 원하는 경우 → Finalize로 이동한다.
- `반복 횟수` >= 5 → Finalize로 이동한다.
- `반복 횟수` < 5 → `반복 횟수` +1 후 Step 1로 돌아간다.
- 사용자가 계속 검토를 원하는 경우 → `반복 횟수` +1 후 Step 1로 돌아간다.

### Finalize

- 사실 오류가 남아있으면 status를 `reviewed`로, 없으면 `completed`로 변경한다.
- status가 `completed`이고 date가 비어있으면 현재 날짜를 기록한다.

## Exceptions

- 노트 파일 경로가 입력되지 않은 경우 : "노트 파일 경로를 입력해주세요." 라는 메시지를 출력하고 종료한다.
- 노트 파일이 존재하지 않는 경우 : "노트 파일이 존재하지 않습니다." 라는 메시지를 출력하고 종료한다.

## Self-Check

- 노트 파일의 오타가 수정되었는가?
- 노트 파일의 사실 오류가 올바르게 제안되었는가?
- 노트 파일의 frontmatter의 `status` 필드가 올바르게 변경되었는가?
- 노트 파일의 frontmatter의 `date` 필드가 올바르게 기록되었는가?
- 최대 5회 이내로 검토가 진행되었는가?
- Finalize가 루프 종료 후 한 번만 실행되었는가?
